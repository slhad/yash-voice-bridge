# Agent Guidelines for yash-voice-bridge

## Essential Commands

- Check specifications: `cat SPECS.md`
- Read project info: `cat README.md`
- Read ongoing work: `cat tmp/ONGOING.md 2>/dev/null || echo "No ongoing work"`
- Run raw mode: `bun run raw`
- Type check: `bun run typecheck`

## Alias

- `[root]`: repository root
- `[tmp]`: local temporary directory for runtime files, logs, recordings, and notes

## tmp/ Is Never Committed

Never commit files under `[tmp]` except the placeholder `tmp/.gitkeep`.

Anything generated by recording, runtime capture, experiments, or local coordination notes should stay untracked.

## Working Rules

1. Read `SPECS.md` before changing behavior.
2. Keep the first milestone narrow: mic capture, local transcription, raw console output.
3. Prefer Bun/TypeScript unless the work item explicitly requires a Rust component.
4. Keep YASH integration outside the capture loop until raw transcription is stable.
5. If you need active local notes, write them in `tmp/ONGOING.md` without committing them.
6. Clean completed `tmp/ONGOING.md` items once they are actually done.

## Architecture Guidance

- This repo is an external sidecar for YASH, not a replacement for YASH IPC.
- YASH should remain the source of truth for action execution and stream/OBS behavior.
- This repo should own:
  - microphone capture
  - transcription
  - raw transcript visibility
  - future IPC client calls

## Validation

Before claiming the raw pipeline works:

- confirm capture starts against the chosen Pulse source
- confirm WAV chunks are created under `[tmp]`
- confirm completed chunks are passed to `whisper-cli`
- confirm recognized text prints to the terminal when the mic picks up speech
